# 设计文档模板（Design Doc Template）

在需要做方案比较、接口设计、数据模型设计时，可参考以下骨架。

## 适用场景
- 任务复杂度较高
- 涉及核心架构、接口协议、数据模型或多方案对比
- 需要将方案沉淀到 `spec/features/<feature-key>/04_design.md`

## 推荐结构
```md
# [Feature Name] Technical Design
> Status: Draft | Date: YYYY-MM-DD

## 1. 概述
- 目标
- 范围
- 非目标

## 2. 方案对比
### 方案 A（推荐）
- 优点：...
- 缺点：...
- 成本：...

### 方案 B
- 优点：...
- 缺点：...
- 成本：...

## 3. 架构与流程
- 组件关系
- 数据流或时序图（可选 Mermaid）

## 4. 数据模型
- 新增/修改字段
- 枚举或状态流转
- 迁移要求

## 5. 接口设计
- Route / Method / Request / Response
- 错误码或失败行为

## 6. 安全与权限
- 鉴权
- 脱敏
- Rate Limit / Abuse 防护

## 7. 验证与风险
- 验证方式
- 风险
- 回滚策略
```

## 使用提示
- 若没有实际对比价值，不要为了形式强行写多方案。
- 只保留与当前任务直接相关的设计内容。
