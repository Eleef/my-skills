# DocOps 拆分输出

此文件夹包含了原始工作流标准的实际拆分，分为：

- `rules/` → 轻量级的常驻指南
- `skills/docops-mode/` → 用于 DocOps 状态/引导/规范到实现的桥接/交付的工作流技能
- `skills/doc-template-router/` → 用于模板路由、标准路由、检查清单和结构化起草的技能

## 预期用途
- 审阅后，将 `rules/DOCOPS_RULES.md` 放入你的持久化项目规则层。
- 将这两个技能作为可重用的、按需调用的工作流组件保留。
- 在技能参考文件中演进模板和路由，而不是让规则层变得臃肿。
- 在将技能移入主技能目录之前，使用 `test_prompts.md` 进行快速手动触发检查。
- 使用 `MIGRATION_GUIDE.md` 了解移动计划和首批测试顺序。

## 为什么要进行此次拆分
- 稳定的约束属于规则。
- 多步骤工作流属于工作流技能。
- 模板路由和检查清单逻辑属于模板技能。
