# 结构化模板笔记 (Structural Template Notes)

每次结构化编辑仅使用一个模板家族。保持结构精简，除非任务确实需要，否则避免加载或重现每一个模板正文。

## PROJECT_INDEX
用于全局文档入口点和导航地图。
包括：
- 项目概览
- 架构一览
- 文档导航
- 新文档存放位置

## PLATFORM_*
用于共享的前端/后端/运维文档以及平台拥有的规范。
保持平台文档的通用性，使其可跨应用重用。
当文档是可重用的平台契约或 Schema 的权威来源时，使用 `docs/platform/<component>/specs/**`。
不要吸收应用特定的细节。

## PLATFORM_STANDARDS
用于应跨应用适用的可重用工程规则、约定、检查清单或政策。
包括：
- 范围
- 规则集或标准
- 何时应用
- 有帮助的示例或反例
- 链接到实施该规则的 ADR、规范或应用文档

## APP_README
用于应用首页。
包括：
- 应用是什么
- 指向词典/功能/指南的快速链接
- 可选的领域术语和高层流程
- 可选的当前状态

## FEATURE_SPEC
用于故事类的功能文档。
包括：
- 目标与非目标
- 用户故事与价值
- UX/UI 行为
- 端到端流程
- 词典引用而非复制的 Schema
- 验收标准
- 根据需要的边界情况和发布说明

## DICTIONARY_API_SCHEMA
用于 API 契约。
保持 Endpoint 标题稳定，并使请求/响应/错误明确化。

## DICTIONARY_DATA_SCHEMA
用于数据和表契约。
保持表标题稳定，并记录目的、列、约束、索引和迁移说明。

## DICTIONARY_UI_SCHEMA
用于 UI 契约文档，如路由、状态机、埋点事件和错误映射。

## DEV_STATUS_*
用于活动任务、待办事项和历史记录更新。
首选微小编辑，并保留顺序/轨迹隔离。

## ISSUE
用于错误报告和事件复盘记录。
保持复现步骤、根本原因、解决方案、验证和回滚易于扫描。

## ADR_*
用于架构决策。
使上下文、决策、后果和参考资料明确化。
